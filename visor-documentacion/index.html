Visor Documentacion

Para este proyecto, mi objetivo inicial era crear un espacio donde se pudieran visualizar documentos de cualquier extensión de manera ordenada y clara, siguiendo una interfaz similar a la de Google Drive y otros servicios de almacenamiento en la nube.

Por ello, lo primero que quise hacer fue crear la aplicación web utilizando un prompt completo y muy específico con todas las funcionalidades que quería implementar para ChatGPT. Una vez generado el programa, procedí a configurar el servidor en la nube, del cual hablaré más adelante.

Este es el prompt que utilicé

Desarrolla una aplicación web responsive y multiplataforma, empaquetada en un único archivo index.html,
que funcione como un mini–Google Drive local. Debe permitir subir, gestionar, previsualizar y organizar
archivos de cualquier extensión, funcionando totalmente offline y sin servidores externos.

La aplicación web que obtuve ya era bastante sólida, pero quise perfeccionar una serie de características que consideraba fundamentales para la experiencia del usuario final.

Solicité que la aplicación permitiera subir carpetas directamente y que la función de crear carpetas, que en ese momento no funcionaba, se corrigiera para que además pudiera arrastrar archivos dentro de ellas o moverlos con un simple clic. También mencioné que el botón para crear nuevas carpetas no parecía hacer nada, y me explicaron que se debía a que la versión anterior utilizaba una ventana emergente del navegador que estaba bloqueada en el entorno de previsualización. Además, pedí que, al subir un archivo que ya existiera, apareciera una ventana como la de Google Drive con opciones para reemplazarlo, cambiarle el nombre o subirlo igualmente. Por último, solicité añadir un sistema de filtrado por nombre, tamaño, fecha de subida o extensión, y señalé que, aunque el botón de ordenación funcionaba, los elementos no se organizaban correctamente.

El resultado final fue este:

Una vez terminada la interfaz, me propuse conectarla a un servidor. Para ello decidí utilizar Firebase, una plataforma de desarrollo creada por Google que ofrece servicios en la nube como bases de datos, almacenamiento de archivos, autenticación y hosting. En mi caso, como solo necesitaba realizar una tarea relativamente sencilla, me limité a utilizar Firestore, la base de datos en la nube de Firebase, y no Firebase Storage.

Antes de enviar el prompt a Gemini, tuve que realizar una serie de ajustes para asegurar que la conexión entre Firebase y la aplicación web funcionara correctamente.

En primer lugar, configuré el acceso como conexión anónima para evitar complicaciones durante el proceso de entrada y asegurar un acceso rápido y sencillo a la aplicación.

Posteriormente, como únicamente quería comprobar la conexión y subir un par de archivos, configuré la base de datos en modo de prueba. Este modo permite que cualquier usuario pueda escribir en la base de datos durante 30 días, lo cual resulta ideal para realizar pruebas rápidas; sin embargo, después es necesario cambiarlo a un modo más seguro.

Una vez realizado todo lo anterior, le proporcioné a Gemini el SDK correspondiente. Un SDK (Software Development Kit) es un conjunto de herramientas, librerías y configuraciones que permiten integrar un servicio —en este caso Firebase— dentro de una aplicación web. Gracias a este SDK pude añadir a la web la opción de subir archivos directamente a la nube.

La aplicación quedo tal que así:

La conexión funcionó a la primera y, además, se añadió un indicador que mostraba si la aplicación estaba conectada correctamente a la nube, así como los posibles errores en caso de que la conexión no pudiera establecerse.

Barra Lateral

  • Contiene el botón principal "Nuevo" para crear carpetas o subir archivos.

  • Navegación principal: "Mi Unidad" (Local), "Papelera", "Nube (Firestore)" y "Estadísticas".

  • Muestra una barra de progreso del almacenamiento local usado.

Main

  • Header: Barra de búsqueda, botón de "Conexión a Firebase", cambio de tema (Claro/Oscuro), opciones de ordenación y cambio de vista (Lista/Cuadrícula).

  • Breadcrumbs: Muestra la ruta actual (ej: Inicio / Carpeta A / Documentos).

  • Contenedor de Archivos: Aquí se renderizan dinámicamente los iconos de archivos y carpetas.

  • Zona de Arrastre (drop-zone): Un área invisible que se activa cuando arrastras archivos desde tu escritorio al navegador.

Modales y Menús Ocultos

  • Ventanas emergentes para: crear/renombrar (input), confirmar borrado, resolver conflictos (si un archivo ya existe), mover archivos y una vista previa para ver imágenes, videos o escuchar audio.

  • Menú contextual (clic derecho) personalizado.

Integración con Firebase (Nube)

  • Configuración: Inicia la conexión con las claves de API proporcionadas.

  • Autenticación: Realiza un Login Anónimo automático para que el usuario pueda guardar datos sin registrarse explícitamente.

  • Subida/Bajada: Convierte los archivos a formato Base64 (texto) para guardarlos dentro de documentos de Firestore (limitado a archivos pequeños, <1MB por la lógica aplicada).

Base de Datos Local (IndexedDB) y Gestión del Estado

  • Clase DB : Crea una base de datos real dentro del navegador del usuario. Esto permite que la aplicación funcione y guarde archivos sin internet (en la sección "Local").
  • Gestiona operaciones CRUD (Crear, Leer, Actualizar, Borrar) para archivos y carpetas.
  • Mantiene variables globales para saber en qué carpeta estás (currentFolderId), si estás viendo la nube o local, el modo de vista (grid/list) y el ordenamiento.

Funciones de Interfaz (UI)

  • render (): Es el corazón visual. Limpia la pantalla y vuelve a dibujar los archivos y carpetas basándose en los datos actuales y los filtros de búsqueda.

  • Navegación: Funciones para entrar en carpetas y actualizar la ruta ("migas de pan").

  • handleFileSelect / Drag & Drop: Detecta cuando el usuario suelta archivos, comprueba si hay conflictos de nombres y los guarda en la base de datos local.

Conclusión

El código crea un aplicación para gestión de documentos. El usuario puede organizar sus archivos localmente en el navegador, ver estadísticas de uso, visualizar contenido multimedia y, opcionalmente, sincronizar o subir archivos específicos a una base de datos en la nube (Firebase Firestore).
