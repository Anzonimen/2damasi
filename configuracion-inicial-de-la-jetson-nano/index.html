Configuración inicial de la Jetson Nano y Preparación e instalación de LM Studio

En este trabajo profundizaremos en la instalación de un potente módulo de computación compacto: la NVIDIA Jetson Orin Nano.

 

La NVIDIA Jetson Orin Nano es un potente módulo de computación compacto diseñado para llevar la inteligencia artificial y la robótica al "borde" (Edge AI). Basada en la arquitectura GPU Ampere, ofrece hasta 40 TOPS de rendimiento, superando hasta en 80 veces a la Jetson Nano original. Es la solución ideal para desarrolladores que necesitan ejecutar modelos de IA modernos y visión por computador con una alta eficiencia energética y un tamaño reducido.

Para la preparación, primero insertamos una tarjeta microSD con el sistema operativo ya grabado y conectamos tanto el teclado como el ratón. Después, nos aseguramos de conectar correctamente el cable de alimentación para darle energía a la placa. Finalmente, conectamos el monitor usando el puerto DisplayPort para poder visualizar el proceso de arranque.

Aceptación de licencias: A continuación, revisamos y aceptamos los términos de licencia de usuario final de NVIDIA marcando la casilla inferior, un paso obligatorio para poder utilizar el software y los drivers del dispositivo.

Selección de idioma: Para comenzar, seleccionamos el idioma 'Español' en la pantalla de bienvenida. Esto asegura que toda la interfaz y los menús de configuración posteriores se muestren en nuestro idioma.

Configuración del teclado: Configuramos la disposición del teclado. Elegimos la opción 'Spanish' para asegurarnos de que la distribución de las teclas (como la ñ o los acentos) coincida con el teclado físico que tenemos conectado.

Selección de red Wi-Fi: En el apartado de red inalámbrica, buscamos y seleccionamos nuestra red de trabajo para conectar el dispositivo a internet.

Creación de Usuario y Credenciales: Configuramos el perfil de usuario principal asignando el identificador 'aulaateca26' tanto para el nombre de usuario como para el nombre del equipo. Establecimos la contraseña y marcamos la opción 'Iniciar sesión automáticamente', lo que nos permitirá acceder directamente al escritorio al encender la Jetson sin necesidad de introducir la clave cada vez.

Configuración de zona horaria: Establecimos nuestra ubicación geográfica seleccionando Madrid en el mapa. Esto ajusta automáticamente la hora y fecha del sistema a nuestra zona horaria local.

Tamaño de partición: Definimos el tamaño de la partición para aplicaciones (APP Partition Size). Aceptamos el valor máximo sugerido por el sistema (117782 MB) para aprovechar todo el almacenamiento disponible en la tarjeta.

Selección de navegador: El asistente nos ofreció la opción de instalar un navegador web. Marcamos 'Install Chromium Browser' para tener una herramienta de acceso a internet lista desde el primer arranque.

Justo después del logo, esperamos unos instantes mientras aparecía esta barra de progreso indicando "Configurando el hardware...", un proceso automático necesario para que el sistema reconozca todos los componentes de la placa.

Tuvimos que esperar brevemente a que se completara esta última barra de progreso, donde el sistema estaba "Applying changes" (aplicando los cambios) y ejecutando tareas de post-instalación antes de mostrarnos el escritorio listo para usar.

Después de completar la configuración inicial y reiniciarse el sistema, vimos esta pantalla de carga con el logotipo de NVIDIA, indicando que la Jetson Orin Nano estaba arrancando el sistema operativo correctamente.

El asistente de bienvenida nos ofreció conectar cuentas en línea (como Google o Microsoft). Decidimos omitir este paso pulsando el botón "Skip" (situado arriba a la derecha) para agilizar el acceso al escritorio.

El sistema nos preguntó si queríamos habilitar "Ubuntu Pro" para soporte extendido y características de seguridad adicionales. Seleccionamos la opción "Skip for now" ya que no es necesario para nuestras prácticas actuales.

En la pantalla de ayuda para mejorar Ubuntu, dejamos marcada la opción "Yes, send system info to Canonical" para enviar datos anónimos del sistema a los desarrolladores y luego pulsamos "Next".

En el apartado de privacidad del asistente de bienvenida, decidimos mantener desactivados los Servicios de Ubicación para no compartir nuestra localización geográfica con las aplicaciones, y pulsamos "Next" para continuar.

Llegamos a la pantalla final del asistente ("You're ready to go!"), donde se nos mostraron algunas aplicaciones populares disponibles para instalar. Hicimos clic en el botón "Done" para cerrar la bienvenida y acceder al escritorio limpio.

Inmediatamente después, el sistema detectó automáticamente que había paquetes desactualizados desde el lanzamiento de la versión. Nos apareció esta ventana avisando de una descarga de 1224 MB y seleccionamos "Install Now" para actualizar el sistema a la última versión disponible.

 Para autorizar la instalación de estas actualizaciones del sistema, se nos solicitó autenticación. Introdujimos la contraseña de nuestro usuario (aulaateca26) y pulsamos "Authenticate" para conceder los permisos de administrador necesarios.

Una vez terminada la actualización crítica de los componentes de NVIDIA, recibimos esta notificación indicando que es necesario reiniciar el equipo ("Reboot is required") para completar la instalación del bootloader y asegurar el correcto funcionamiento de la placa.

Observamos la barra de progreso mientras el "Software Updater" descargaba e instalaba automáticamente los paquetes necesarios, incluyendo librerías críticas como nvidia-l4t-vulkan y componentes de Docker, fundamentales para ejecutar contenedores de IA.

Al finalizar la instalación de los paquetes, el sistema nos mostró un aviso indicando que "The computer needs to restart...". Hicimos clic en el botón "Restart Now" para aplicar los cambios y permitir que el firmware de la placa base se actualizara correctamente.

 Durante el reinicio, la pantalla mostró el logotipo de NVIDIA con una barra de carga de texto ("Update Progress"). Aquí verificamos que se estaba actualizando el firmware del sistema (Bootloader) a la versión 36.4.3, un paso crítico que no debemos interrumpir bajo ningún concepto.

Una vez reiniciado el equipo y de vuelta en el escritorio, accedimos al menú de Configuración > Bluetooth. Activamos el interruptor para buscar dispositivos cercanos y conectar nuestros periféricos inalámbricos, detectando varios equipos en el aula.

Seleccionamos nuestro ratón inalámbrico y teclado de la lista y confirmamos el emparejamiento. La ventana nos mostró el estado "Paired: Yes", confirmando que el dispositivo estaba conectado y listo para trabajar de manera más cómoda sin cables.

Finalmente, nuestra Jetson Nano está oficialmente configurada y lista para ejecutar cualquier tarea que necesitemos.

Preparación e instalación de LM Studio

Configurar el disco NVMe como dispositivo de almacenamiento en la Jetson

Buscar Jetson Lab en el navegador para acceder a los tutoriales y guías oficiales necesarios para configurar la Jetson y completar correctamente el proceso de instalación.

Acceder a Explorar tutoriales porque allí se encuentran las guías paso a paso que explican cómo realizar correctamente la configuración de la Jetson y las tareas necesarias para el proyecto.

Entrar en Jetson Setup Guide porque es la guía oficial que explica el proceso de configuración inicial de la Jetson, incluyendo la detección del hardware, la preparación del sistema y los pasos necesarios antes de instalar software adicional.

(La parte de Physical Installation ya fue realizada previamente, por lo que podemos continuar directamente con la configuración lógica y del software.)

Abrir la terminal con Ctrl + Alt + T porque necesitamos un entorno de línea de comandos para ejecutar las instrucciones técnicas que permiten comprobar el hardware, montar el disco y continuar con la instalación.

Ejecutar el comando lspci porque este comando muestra una lista de los dispositivos PCI conectados, y nos permite verificar que la Jetson reconoce físicamente el disco NVMe o cualquier otro componente relevante antes de seguir con la configuración.

Ejecutar lsblk porque este comando muestra todos los dispositivos de almacenamiento conectados al sistema, incluyendo discos duros, particiones y unidades NVMe, lo que nos permite confirmar que el disco está detectado correctamente.

Aquí comprobamos si el disco NVMe aparece listado porque necesitamos asegurarnos de que el sistema lo reconoce antes de formatearlo o montarlo.

Para montarlo, seguimos los pasos indicados en la documentación porque allí se explica de forma detallada qué hace cada comando, garantizando que el procedimiento se realiza correctamente y evitando errores que puedan afectar al sistema o al disco.

Mientras montamos el disco, podemos ir instalando LM Studio porque ambos procesos son independientes entre sí y así aprovechamos mejor el tiempo, avanzando en la configuración del sistema sin necesidad de esperar a que finalice una tarea para comenzar la otra.


En la terminal, ejecutar estos comandos en orden porque cada uno cumple una función específica y deben realizarse secuencialmente para preparar y montar correctamente el disco NVMe:

  • sudo mkfs.ext4 /dev/nvme0n1
    Porque este comando formatea el disco NVMe con el sistema de archivos EXT4, que es el estándar en Linux y necesario para poder usarlo como almacenamiento.

  • sudo mkdir /ssd
    Porque necesitamos crear una carpeta (punto de montaje) donde el sistema pueda “conectar” el disco para acceder a sus archivos.

  • sudo mount /dev/nvme0n1 /ssd
    Porque este comando monta el disco NVMe dentro de la carpeta /ssd, permitiendo que el sistema pueda leer y escribir en él.

  • lsblk -f
    Porque nos permite verificar que el disco está correctamente formateado, montado y con el sistema de archivos adecuado, asegurando que los pasos anteriores se ejecutaron correctamente.

Hacemos un sudo apt install nano porque necesitamos instalar el editor de texto nano, que nos permitirá modificar archivos del sistema de forma sencilla desde la terminal.

Ejecutamos sudo nano /etc/fstab porque este comando abre el archivo fstab, que es donde se configuran los puntos de montaje automáticos del sistema.

En el archivo fstab añadimos la línea correspondiente para montar el disco automáticamente porque así el NVMe se montará cada vez que encendamos la Jetson, evitando tener que hacerlo manualmente en cada inicio y garantizando que el sistema siempre tenga acceso al disco.

Instalar libfuse 2 y libfuse 3 usando los comandos que aparecen en las capturas porque LM Studio necesita estas bibliotecas para poder ejecutar correctamente sus componentes y manejar sistemas de archivos en modo usuario.

Para libfuse 2 instalamos libfuse2 porque es la versión antigua de la librería que algunos programas todavía requieren para funcionar.

Para libfuse 3 instalamos fuse3 porque es la versión más reciente y necesaria para aplicaciones modernas que gestionan montajes o archivos empaquetados de forma segura.

Dar permisos de ejecución a apiimagen porque el sistema necesita autorización para poder ejecutar ese archivo como un programa. Si no se otorgan estos permisos, el instalador no podrá iniciarse.

Proceder con su instalación porque apiimagen forma parte de los componentes necesarios para que LM Studio funcione correctamente en la Jetson, y sin instalarlo no se completaría la configuración del entorno.

Abrimos LM Studio porque necesitamos iniciar la aplicación para comprobar que la instalación se realizó correctamente y que el programa arranca sin errores.

Verificamos que funciona porque es importante asegurarnos de que la interfaz, los menús y las funciones principales están operativas antes de continuar con la configuración.

Seleccionamos el icono de la lupa en la barra lateral izquierda para instalar modelos porque esa sección es donde LM Studio permite buscar, visualizar y descargar los modelos de inteligencia artificial que utilizará la aplicación.

Instalamos el modelo qwen3-vl-4b porque es el modelo de inteligencia artificial que vamos a utilizar en LM Studio y necesitamos descargarlo para poder ejecutar consultas y pruebas.

En la ficha del modelo aparece su descripción porque LM Studio muestra información técnica que ayuda a entender qué tipo de modelo estamos instalando, sus capacidades y sus requisitos.

Para instalarlo, pulsamos el botón verde Download porque es la acción que inicia la descarga y configuración automática del modelo dentro de LM Studio.

En params, 4B significa 4 Billions (4 mil millones) porque ese número indica la cantidad de parámetros que tiene el modelo, que son los componentes internos que utiliza para aprender y generar respuestas.

Estos parámetros representan las conexiones entre neuronas del modelo porque, en una red neuronal, cada parámetro corresponde a un peso que define cómo interactúan las neuronas entre sí; cuantas más conexiones haya, más complejo y capaz es el modelo.

Creamos un nuevo chat pulsando el + en la barra lateral porque necesitamos iniciar una conversación desde cero para probar el modelo recién instalado y verificar que LM Studio está generando respuestas correctamente.

Probamos escribiendo "hola" porque es una forma sencilla y rápida de verificar que el modelo responde y que la comunicación entre LM Studio y el modelo está funcionando sin errores.

Comprobamos el número de tokens por segundo porque esta métrica indica la velocidad a la que el modelo genera texto, lo cual es útil para evaluar su rendimiento y asegurarnos de que está operando de manera fluida en la Jetson.

 

Finalmente, tendríamos LM Studio instalado y funcionando de manera óptima.
